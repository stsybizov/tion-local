# Tion Local

[🇬🇧 English](README.md) · 🇷🇺 Русский

Локальное управление бризером **Tion 4S** в [Home Assistant](https://www.home-assistant.io/) по **Bluetooth (BLE)** — без облака, без модуля MagicAir и без учётной записи Tion. Home Assistant общается с прибором напрямую по Bluetooth Low Energy.

> [!WARNING]
> Независимый общественный проект. **Не связан с Tion / ООО «ТИОН», не одобрен и не поддерживается ими.** Бризер — приточное устройство, а не обогреватель: не используйте его для отопления. Всё, что вы делаете, — на ваш страх и риск.

Это самостоятельный форк, сфокусированный на **Tion 4S**: расширенная телеметрия, нативное турбо и работа с расписанием прибора — реверс-инжиниринг поверх исходной интеграции (см. [Благодарности](#благодарности)). Код для S3/Lite из исходного проекта сохранён, но активно развивается и тестируется здесь только 4S.

## Возможности

- **Климат** — вкл/выкл, `вентиляция` / `обогрев`, целевая температура (0–25 °C), предустановки (Обычный / Турбо / Сон / Не дома)
- **Вентилятор** — вкл/выкл + 6 скоростей и отдельный селектор **скорости** (1–6)
- **Забор воздуха** — улица / рециркуляция
- **Переключатели** — звук, подсветка и «доступ для приложения» (освобождает BLE, чтобы подключилось приложение Tion)
- **Нативное турбо** — собственный таймер прибора (5 / 10 / 15 мин) через кнопку + селектор длительности; авто-возврат к прежней скорости
- **Расписание прибора** — чтение и запись всех 12 таймеров (6 расписаний старт/стоп) по BLE, сервис `write_schedule` и опциональный редактор на дашборде
- **Сенсоры** — температуры входа/выхода, текущая скорость, ресурс фильтра, прошивка, температуры плат, **мощность нагревателя / вентилятора / полная**, **энергопотребление** (кВт·ч, для Energy-дашборда), **производительность** (м³/ч), **прошло воздуха** (м³)
- Бинарный сенсор **«Проблема»** с **расшифровкой кодов** ошибок/предупреждений (EC01–EC11, WS01–WS06)
- Надёжный слой BLE (bleak-retry-connector), перезагрузка из UI, неблокирующий старт

## Требования

- Home Assistant с работающей интеграцией **Bluetooth** (встроенный адаптер, USB-донгл или **ESP32 `bluetooth_proxy`** рядом с бризером — рекомендуется, если прибор далеко от хоста)
- Бризер **Tion 4S**
- Приложение Tion должно быть **отключено**, пока подключён Home Assistant — BLE допускает только одно соединение

## Установка

### HACS (рекомендуется)

1. HACS → ⋮ (вверху справа) → **Пользовательские репозитории**
2. Добавьте `https://github.com/stsybizov/tion-local`, категория **Интеграция**
3. Найдите **Tion Local** в HACS и установите
4. Перезапустите Home Assistant

### Вручную

Скопируйте `custom_components/ha_tion_btle/` в папку `config/custom_components/` вашего Home Assistant и перезапустите.

## Настройка

1. Настройки → **Устройства и службы** → **Добавить интеграцию** → найдите **Tion Local**
2. Заполните поля:
   - **Модель** — `S4`
   - **MAC** — Bluetooth-адрес бризера, `ВЕРХНИЙ` регистр с двоеточиями, напр. `AA:BB:CC:DD:EE:FF`
   - **Сопряжение** — см. примечание ниже
3. Следуйте подсказкам и завершите.

### Примечание о сопряжении

4S использует BLE-сопряжение *Just Works* с очень коротким окном. Если шаг сопряжения в мастере падает с ошибкой аутентификации, выполните привязку **вне HA** один раз (например, `bluetoothctl pair <MAC>` на хосте, пока бризер в режиме сопряжения), а затем добавьте интеграцию **с выключенным сопряжением** — Home Assistant подключится по существующей привязке ОС.

## Сущности

| Сущность | Примечание |
|---|---|
| `climate.tion_4s` | Основное управление: режим, температура, скорость, предустановки |
| `fan.…_fan_speed` | Вкл/выкл + 6 скоростей |
| `select.tion_4s_air_mode` | Улица / Рециркуляция |
| `select.tion_4s_speed` | Скорость 1–6 |
| `select.tion_4s_boost_duration` | Длительность турбо 5 / 10 / 15 мин |
| `number.tion_4s_target_temperature` | Целевая температура, 0–25 °C |
| `switch.tion_4s_sound` / `…_backlight` | Звук / подсветка |
| `switch.tion_4s_app_access_release_ble` | Освободить BLE для приложения |
| `button.tion_4s_boost` | Запуск нативного турбо |
| `button.tion_4s_reset_filter_life` | Сброс счётчика фильтра (подтверждение двойным нажатием) |
| `binary_sensor.tion_4s_problem` | Вкл при errors≠0; атрибут `problems` со списком кодов |
| `sensor.tion_4s_*` | Температуры, ресурс фильтра, прошивка, мощность/энергия, производительность, прошло воздуха, расписание |

## Сервисы

| Сервис | Назначение |
|---|---|
| `ha_tion_btle.write_schedule` | Записать одно из 6 расписаний (дни, старт/стоп, скорость, режим, подогрев) — пишет оба слота старт+стоп |
| `ha_tion_btle.write_timer` | Записать один сырой слот таймера (0–11) — для продвинутых |
| `ha_tion_btle.refresh_schedule` | Перечитать все 12 таймеров с прибора |
| `ha_tion_btle.set_turbo` | Запустить нативное турбо на N секунд |
| `ha_tion_btle.set_timers_enabled` | Включить/выключить все таймеры расписания (служебный) |
| `ha_tion_btle.read_turbo` | Диагностика: состояние турбо + сырой state-фрейм (возвращает данные) |
| `ha_tion_btle.set_air_source` | Задать забор воздуха на климат-сущности |

Стандартные сервисы `climate.*`, `fan.*`, `select.*`, `switch.*`, `number.*` и `button.*` работают со всеми сущностями выше — всё автоматизируется.

### Редактор расписания (опционально)

Расписание прибора (день/ночь, скорость, режим, подогрев) можно редактировать с дашборда через input-хелперы + небольшой скрипт, вызывающий `write_schedule`. Готовый пример — в [`docs/schedule-editor.md`](docs/schedule-editor.md).

> Одно «расписание» Tion хранится как пара слотов (старт + стоп). Интеграция показывает только *активные* расписания (таймер активен, если он включён **и** выбран хотя бы один день недели — приложение «удаляет» расписание, обнуляя дни).

## Заметки про турбо / BLE

- Нативное турбо работает по таймеру прибора и возвращается к прежней (расписанной) скорости; совместимо с расписанием прибора.
- BLE — **одно соединение**: пока подключено приложение Tion, Home Assistant не может работать с бризером, и наоборот. Используйте переключатель «доступ для приложения», чтобы передать линк.
- Слабый BLE-линк иногда теряет опрос; **ESP32 bluetooth proxy** рядом с бризером делает связь стабильной.

## Решение проблем

- **Сущности `unavailable` после рестарта** — бризер переподключается на ближайшем опросе; если не проходит — проверьте, включён ли прибор и в зоне ли действия (или добавьте Bluetooth-прокси).
- **Не получается отредактировать расписание из приложения Tion** — сначала включите `switch.tion_4s_app_access_release_ble`, чтобы HA отпустил линк.
- **Применить изменения без полного рестарта** — используйте **Перезагрузить** в меню ⋮ интеграции.
- Отладочный лог:
  ```yaml
  logger:
    default: warning
    logs:
      custom_components.ha_tion_btle: debug
  ```
